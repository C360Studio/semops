# Command Lifecycle Fixture Review

Date: 2026-06-23

Decision: accept a SemOps-authored command lifecycle JSONL fixture as derived story data for command-policy replay.

The fixture covers three deterministic command outcomes: a local MAVLink route request that progresses through native
accepted/executing readback, local cancellation, and native cancelled readback; a federated survey request that is
accepted and then times out; and a stale federated request that expires before acceptance. The committed file is
generated by `internal/projectors/command.LifecycleFixtureRecords`, checked byte-for-byte by the command projector
tests, and listed in `fixtures/manifest.json` with SHA-256 and byte size.

Adversarial findings:

- The fixture is synthetic SemOps story data. It is not captured CS API traffic, MAVLink COMMAND_LONG/COMMAND_ACK
  traffic, TAK/SAPIENT tasking traffic, DJI command telemetry, or hardware evidence.
- Replay proves the pure command-intent policy path can apply requested, accepted, executing, cancel-requested,
  cancelled, timeout, and expired lifecycle states through existing projector/reconciliation helpers.
- Replay does not prove CS API Command or ControlStream request handling, a hosted command scheduler, native command
  transmission, native cancellation delivery, radio safety interlocks, authority delegation, or operator UI semantics.
- Deadline evidence now carries an optional native command ID for replay/scheduler routing and rejects mismatches when
  present, matching the existing native-status evidence guard.
- The fixture should be cited as portable demo/regression evidence only. Live command/control remains blocked until
  the CS API/UI/native-driver boundary, stale-command rejection, local override, safety interlocks, and asynchronous
  status feedback are reviewed against live or simulator-backed sources.
