# KLV One-Command Stack Smoke Review

Date: 2026-06-22
Scope: opt-in hosted KLV deterministic MPEG-TS smoke through the local COP stack.

## Decision

Accept the opt-in KLV stack smoke as the first end-to-end hosted proof for deterministic KLV media-derived state.

`SEMOPS_COP_SMOKE_KLV_ENABLED=true bash scripts/cop-stack-smoke.sh` generates a tiny MPEG-TS fixture from the
committed MISB ST 0601 truth JSON, builds the `media-tools` container target with FFmpeg, enables the KLV
media-reference input -> demux -> decode -> projector flow, and verifies Caddy-routed COP snapshot readback plus
shared Prometheus/runtime flow samples.

KLV remains disabled by default, and the default Compose build target remains the production image. The FFmpeg-bearing
`media-tools` image target is for local deterministic media proof and future media-runtime investigation.

## Adversarial Notes

### High: One deterministic clip is not live media support

The smoke proves a generated local MPEG-TS file through the hosted component chain. It does not prove object-storage
materialization, directory watching, RTSP/RTP ingest, video serving, cache policy, or long-running media behavior.

Resolution: keep KLV opt-in and keep live media ingress under a later media-stack review.

### High: This is still not STANAG 4609 conformance

The fixture carries a supported MISB ST 0601 subset generated by SemOps. It is strong deterministic engineering
evidence for that subset, but it is not an official STANAG 4609 validator or lab result.

Resolution: continue using "engineering support for the tested subset" language and keep formal conformance as a
separate funded path.

### Medium: FFmpeg is an image/profile cost

The `media-tools` target carries FFmpeg and its runtime dependencies, while the production image remains distroless.
This keeps the proof path useful without making the default SemOps service heavier.

Resolution: revisit sidecar, SemSource media service, static FFmpeg, GStreamer, or MediaMTX only when real media
traffic or deployment constraints force that decision.
